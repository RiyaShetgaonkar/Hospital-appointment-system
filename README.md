Smart Digital Queue & Token System Reducing waiting time and overcrowding for patients in government hospitals.

Problem Government hospitals often have long, chaotic queues. Elderly and rural patients wait for hours in overcrowded conditions just to see a doctor.

Solution A digital queue system where patients can: Sign in with email View clinics and join a queue remotely Get a token number and real-time ETA Arrive at the hospital just in time Hospital staff can manage queues efficiently via a simple dashboard, calling the next patient and updating token status.

Key Features Remote token generation for multiple clinics Real-time queue position and ETA Staff dashboard to call, skip, or complete tokens Reduced overcrowding and waiting time

Impact Less waiting and standing time Reduced overcrowding Easier access for elderly and rural patients Less stress for hospital staff

Tech Used Firebase Authentication – Email/password sign-in for secure patient access Firestore – Stores patient data, clinic info, tokens, and provides live updates Node.js – Backend framework to handle API requests, queue logic, and integration with Firebase
